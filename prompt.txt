Production pipeline requirements:
- Process the manifest sequentially in terms of global task ownership, but allow two workers to run different tasks at the same time.
- Convert each DAV file to 15 fps using ffmpeg.
- Split each converted MP4 into 200-frame chunks to stay within Tesla T4 memory limits.
- Run SAM3 inference for each prompt.
- Save results to MEGA under `/SAM3/results`.
- Maintain `/SAM3/session.json` and `/SAM3/prompt.txt` in MEGA.
- Ensure a fresh future agent can resume from the saved MEGA state alone.

Session and resumability requirements:
- `session.json` must contain enough information to know:
  - manifest index
  - filename
  - source URL / id
  - prompt completion state
  - failed prompts
  - uploaded result paths
  - currently claimed task
  - summary counts
  - stop reason
- `prompt.txt` in MEGA must contain a short resume handoff for the next agent.
- Every meaningful state change must be synced back to MEGA.

Two-worker production design:
- Worker A:
  - `CUDA_VISIBLE_DEVICES=0`
  - claims one available task from session state
- Worker B:
  - `CUDA_VISIBLE_DEVICES=1`
  - claims a different available task
- Task claiming must be atomic enough that both workers cannot take the same unit of work.
- Acceptable approaches:
  - file lock around session mutation
  - lock file in `/kaggle/working/SAM3`
  - another safe atomic claim/update mechanism

Task granularity:
- Recommended claim unit:
  - one manifest item plus one prompt
  - or one manifest item with internal prompt ownership tracked carefully
- Avoid duplicate uploads and duplicate work.

Cleanup requirements:
- After successful upload, remove local temporary files where safe:
  - downloaded `.dav`
  - converted `.mp4`
  - temporary chunk files
- If low disk space is detected, stop cleanly and checkpoint state.

Benchmark facts you should trust:
- Single-GPU runtime for one 200-frame chunk was about `663.52s` to `669.89s`.
- Two independent single-GPU workers processed two 200-frame chunks in `676.31s` wall-clock.
- This is about `1.97x` throughput improvement over one GPU running the two chunks sequentially.
- Built-in SAM3 multi-GPU was not a good production option for this workload.
